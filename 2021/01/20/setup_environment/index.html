Git commit và staging area

Tracked vs untracked

Một tập tin có 2 trạng thái chính là:

  • Tracked:
    • Unmodified: chưa chỉnh sửa
    • Modified: đã chỉnh sửa
    • Staged: sẵn sàng commit
  • Untracked: là tập tin không muốn làm việc trong git

Lệnh git add <file> đưa 1 tập tin từ untracked thành tracked Chuyển từ tracked về untracked: rm <file> Xoá hẳn 1 file: git rm -f <file> Bỏ tập tin ra khỏi staging aread: git reset HEAD <file>

Git tag

Trong git có 2 kiểu tag là: - Lightweight tag: đánh dấu snapshot của commit - Annotated tag: có thể đặt tiêu đề cho tag, có các thông tin về người tag, ngày tag

Git sẽ đánh dấu lệnh commit cuối cùng vào tag

Lightweight tag

Tạo 1 tag: git tag <name> Xem danh sách tag: git tag Xem tag gắn với commit: git show <tag>

Annotated tag

Tạo 1 tag: git tag -a <name> -m <title>

Branch tag

Tạo một branch mới với dữ liệu cmoot trên tag: git checkout -b <branch name> <tag>

push vs pull vs clone

  • git clone: Sao chép toàn bộ thiết lập và dữ liệu của master branch
  • git pull: lấy dữ liệu từ remote repository và gộp vào branch hiện tại đang làm việc tự đông
  • git fetch: Lấy dữ liệu từ remote repository và gộp thủ công vào branch nào đó với lệnh git merge <branch name>

Create multiple virtual environment with different Python version

  1. Open "Anaconda Prompt" and type

conda create -n <ENV_NAME> python=<VERSION> <LIBRARY1> <LIBRARY2>

  1. Activate your env: activate <ENV_NAME>
  2. Install jupyter notebook and register it on kernelspec (to use all notebook kernels at once place) in one shot: python -m ipykernel install --user --name <NAME> --display-name <DISPLAY NAME ON JUPYTER KERNEL LIST>
  3. Install other libraries using PIP
